# UI and UX Standards

## Design Philosophy

- **Modern and Premium**: Clean typography, strong whitespace, subtle animations.
- **Interactive**: Hover, active, and focus states are mandatory on all interactive elements.
- **Mobile First**: Design for small screens first, then scale up.

## Design Tokens (Single Source of Truth)

> Definitions: `app/styles/themes/_shared/base.css`; per-theme overrides in `app/styles/themes/`.

Theme-aware colors MUST use the Vikini token vocabulary via Tailwind's arbitrary syntax:

| Purpose          | Token                                                                 | Example class                         |
| ---------------- | --------------------------------------------------------------------- | ------------------------------------- |
| Page surface     | `--surface`, `--surface-muted`, `--surface-elevated`                  | `bg-(--surface-elevated)`             |
| Text             | `--text-primary`, `--text-secondary`                                  | `text-(--text-primary)`               |
| Controls         | `--control-bg`, `--control-bg-hover`, `--control-border`              | `bg-(--control-bg)`                   |
| Border / divider | `--border`                                                            | `border-(--border)`                   |
| Brand / CTA      | `--accent`, `--accent-foreground`                                     | `bg-(--accent)`                       |
| State            | `--danger` (`--danger-hover`/`-foreground`), `--success`, `--warning` | `text-(--danger)`                     |
| Shared           | `--ring`, `--radius`, `--overlay`                                     | `ring-(--ring)`, `rounded-(--radius)` |

<important>
Tailwind v4 here has NO `@config`, so `tailwind.config.ts` color names are INERT.
BANNED (render to empty styles): `bg-primary`, `bg-destructive`, `bg-background`,
`bg-popover`, `text-muted-foreground`, `border-input`, `ring-ring`, `bg-card`, `bg-muted`.
Use the `bg-(--token)` arbitrary syntax in the table above instead.
</important>

## Anti-Patterns (BANNED)

- Fixed widths like `w-[500px]`. Use `w-full max-w-lg` instead.
- Heavy dark borders. Use `border-(--border)` (subtle) instead.
- Raw color classes like `bg-blue-500` / hardcoded hex (`#020617`) / `neutral-*` for theme
  surfaces. Use the design tokens above so all 15 themes stay correct.
- Dead shadcn token classes (see the `<important>` block above).
- Arbitrary z-index like `z-[9999]`. Use configured z-indices.
- Inline styles `style={{}}`. Use Tailwind utility classes.
- Sub-12px font sizes (`text-[10px]`, `text-[11px]`) for real labels/buttons. Minimum `text-xs` (12px).
- Hand-rolled modal `div`s (`fixed inset-0 …`). Use the `Dialog` primitive (focus-trap + ESC).
- Custom SVG icons. Use Lucide React exclusively.
- `max-h-[2000px]` or similar CSS hacks for collapse animation. Use Framer Motion `height: "auto"`.
- Fixed-width dropdowns on mobile (e.g., `w-[350px]`). Use `w-[min(350px,calc(100vw-2rem))]`.

## Component Usage

- **Shadcn/UI**: Check `components/ui/` first before creating new primitives. The primitives are
  themed with Vikini tokens — extend them, don't fork them.
- **Loading**: Use the `Skeleton` primitive (`components/ui/skeleton.tsx`) for content placeholders.
- **Lucide Icons**: The only icon set allowed. Icon-only buttons MUST have an `aria-label`.
- **Framer Motion**: Use for page transitions, modal animations, and micro-interactions. Keep animations under 300ms for UI feedback, up to 500ms for transitions.
  Import constants from `lib/utils/motion.ts` (`DURATION`, `EASE`) — do NOT hardcode durations/easings per-component.

## Accessibility (MANDATORY)

- All interactive elements need a visible focus state: `focus-visible:ring-2 focus-visible:ring-(--ring)`.
- Dialogs/menus must use Radix primitives (they provide focus-trap, ESC, and ARIA roles).
- Do not convey state by color alone; pair with text or icon.
- Use native `<button>` elements for clickable actions. BANNED: `<div role="button">` or `<span onClick>`.
- Toggle buttons must have `aria-pressed={isActive}`. Collapsible sections must have `aria-expanded={isOpen}`.
- Scrollable message/log areas should have `role="log"` and `aria-live="polite"`.
- Sidebar sections should include `<h2 className="sr-only">` for screen reader navigation structure.

## Theme Integration

- Themes are located in `app/styles/themes/` (focus / glassmorphism / ra2 groups).
- Use CSS variable tokens for theme-aware colors, never hardcoded values, so a component works
  across every theme (and any future Light theme — token infra exists in `base.css`).

## Notifications and Feedback

- Use `useToastStore` / `toast` (`lib/store/toastStore.ts`) for non-blocking feedback.
- Use the imperative `confirm()` (`lib/store/confirmStore.ts`) for confirmations:
  `if (await confirm({ title, variant: "danger" })) { … }`.
- BANNED: Native `alert()` / `confirm()` and hand-rolled confirm modals in feature components.

## Mobile / Responsive

- Do not duplicate controls between mobile and desktop layouts. If a control exists in the mobile
  Settings menu, hide the desktop equivalent with `hidden md:flex`.
- Dropdown panels must not exceed viewport width. Use `min()` for width constraints.
